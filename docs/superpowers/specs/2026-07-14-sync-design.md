# РОЙ — синхронизация между устройствами по коду

Дата: 2026-07-14

## Цель
Заметки, добавленные на одном устройстве, автоматически появляются на других
устройствах пользователя — без регистраций и без настройки бэкенда.

## Не-цели (YAGNI)
- Аккаунты / вход по Google.
- Шифрование содержимого.
- Реалтайм-курсоры, presence, совместное редактирование.
- Разрешение сложных конфликтов сверх «побеждает свежее по времени».

## Главный приоритет: заметки не теряются
Инварианты безопасности данных (важнее любой фичи):

1. **localStorage — источник истины на каждом устройстве.** Синхронизация — надстройка.
2. **Синхронизация только аддитивна к локальным данным:** приходящее из облака может
   ДОБАВИТь заметку или ОБНОВИТь её на строго более свежую версию (по `updatedAt`);
   оно НИКОГДА не удаляет локальную заметку.
3. **Пустой / сбойный / нераспарсиваемый ответ из облака не меняет локальные данные.**
4. **Удаление — мягкое (tombstone):** заметка помечается `deleted:true`, `updatedAt=now`
   и остаётся в хранилище (скрыта из UI, восстановима). Физически не стирается.
5. **Снимок «до синхронизации»:** при первом включении/подключении кода локальное
   состояние сохраняется в `roi-ideas-presync-<ts>` для отката.
6. **Входящие данные проходят через `sanitizeIdea`** — мусор не ломает и не портит стор.
7. **Экспорт/Импорт остаётся** как надёжный ручной бэкап-план.

## Архитектура
- **Транспорт:** GunDB (децентрализованная БД поверх публичных relay-узлов). Ноль
  настройки, произвольные ключи, автоматическое слияние. Грузится **лениво с CDN**
  только когда пользователь включает синхронизацию (база приложения остаётся лёгкой
  и офлайновой).
- **Ключ синхронизации:** секретный код пользователя → `SHA-256` → hex → узел
  `roy-sync-v1/<hash>` в Gun. Сырой код не фигурирует в пути.
- **Хранение в Gun:** на каждую заметку узел `node.get(id).put({ j: JSON.stringify(idea), u: updatedAt })`.
  Идея сериализуется в строку — обходим ограничения Gun на массивы/вложенность,
  полный контроль над слиянием на нашей стороне.

## Модель данных
Добавляется поле `updatedAt` (число, мс). Проставляется при создании, ИИ-разборе,
смене категории, удалении. Плюс мягкое удаление через `deleted:true`.
`sanitizeIdea` расширяется полями `updatedAt`, `deleted`.

## Слияние (merge)
Для входящей заметки `r` с локальной `l` (по `id`):
- нет `l` → добавить `r`;
- есть `l` и `r.updatedAt > l.updatedAt` → заменить поля `l` на `r` (в т.ч. флаг `deleted`);
- иначе → оставить `l`.
Никогда не удаляем `l`. Локальный `push`: пишем изменённую заметку в Gun. При
подключении и при фокусе окна — до-отправляем ВСЕ локальные заметки (реконсиляция,
устойчивость к «потерянным» промежуточным апдейтам).

## Отображение
Ввести `live = ideas.filter(i => !i.deleted)`. Использовать в `filtered()`,
счётчиках чипов, `allTags()`, графе. Удаление в шторке помечает `deleted`, не режет массив.

## UI (⚙ Настройки → новый блок «Синхронизация»)
- Кнопка «Включить». Показ сгенерированного кода + «Скопировать».
- Поле «вставить код» + «Подключить» (для второго устройства).
- Статус: включено / код / «Отключить».
- Виден только в standalone-режиме (не в прототипе).

## Режимы отказа и запасной план
- Публичные relay могут тормозить/лежать → синхронизация «подтормаживает», но
  локальные данные целы; при восстановлении связи устройства до-синхронизируются.
- CDN/Gun недоступен → синхронизация не включится, приложение работает как раньше.
- Запасной план всегда: ручной Экспорт/Импорт.

## Тестирование
- Юнит-тест merge в Node: union по id; LWW по updatedAt; пустой remote не удаляет;
  tombstone-восстановимость.
- Браузер: включение синхронизации грузит Gun без ошибок; пуш без исключений;
  инвариант «пустой remote не стирает локальное».
- Кросс-девайс (две вкладки с одним кодом) — по возможности; финальную проверку на
  реальных телефон+комп делает пользователь.
